## 👻 Ghostty

[↑ Back to Contents](#-contents)

**Role:** Backup terminal emulator.

**Config file:** `dotfiles/ghostty/config` in the repo — symlinked to `~/.config/ghostty/config` for all users via Home Manager.

### Key settings

| Setting | Value | Why |
|---------|-------|-----|
| Font | JetBrainsMono Nerd Font Propo, size 10 | Nerd font glyphs for shell and editor icons |
| Opacity | 0.95 | Slight transparency |
| Blur | 20 | Background blur effect on COSMIC |
| `clipboard-paste-protection` | false | No confirmation prompt when pasting |
| `config-file = ~/.config/ghostty/colors` | dynamic | matugen writes colors here on wallpaper change |

### Custom shaders

Two cursor shaders are active — defined in the config and sourced from `dotfiles/ghostty/shaders/`:

- `cursor_blaze_tapered.glsl` — tapered flame effect on the cursor
- `cursor_smear_fade.glsl` — smear/fade trail on cursor movement

### Colors

Colors come from matugen. When the wallpaper changes, matugen regenerates `~/.config/ghostty/colors` and Ghostty picks it up automatically via the `config-file` directive — no restart needed.

If colors look wrong or out of date:

```bash
cat ~/.config/ghostty/colors                     # see what matugen last generated
cat ~/.local/share/last-matugen-wallpaper         # see which wallpaper triggered the last run
systemctl --user restart wallpaper-color-sync    # force a color refresh
```

## 📝 Neovim

[↑ Back to Contents](#-contents)

**Role:** Primary code editor for linuxury — configured as a full IDE.

**Config location:** `dotfiles/nvim/` — symlinked to `~/.config/nvim/` by Home Manager. All plugins and tools are installed via Nix packages; nothing is managed by lazy.nvim or Mason.

---

### Layout

[↑ Neovim](#neovim)

When the first file opens in a session, the layout auto-configures itself via an autocmd in `autocmds.lua`:

```
┌─────────────┬────────────────────────────┬──────────────────┐
│             │                            │                  │
│  Neo-tree   │         Editor             │   Claude Code    │
│  (sidebar)  │         (center)           │   (right panel)  │
│             │                            │                  │
├─────────────┴────────────────────────────┴──────────────────┤
│                    Terminal (on demand, Ctrl+\)              │
└─────────────────────────────────────────────────────────────┘
```

The autocmd fires once per session on the first `BufReadPost` event — it opens Neo-tree, opens the Claude Code panel to the right, then returns focus to the editor. The terminal is not open by default; toggle it with `Ctrl+\`.

### Options

[↑ Neovim](#neovim)

Set in `dotfiles/nvim/lua/options.lua`. Notable non-defaults:

| Option | Value | Why |
|--------|-------|-----|
| `number` + `relativenumber` | true | Absolute line number on cursor row, relative elsewhere |
| `scrolloff` | 8 | Keep 8 lines visible above/below cursor before scrolling |
| `clipboard` | unnamedplus | Yank/paste goes to the OS clipboard automatically |
| `undofile` | true | Persistent undo across sessions (`~/.local/state/nvim/undo/`) |
| `swapfile` / `backup` | false | Disabled — rely on undo history and git instead |
| `wrap` | false | No line wrapping by default (toggleable with `<leader>tw`) |
| `foldmethod` | expr (treesitter) | Folds defined by syntax, disabled by default — `zR` opens all |
| `pumblend` / `winblend` | 15 | Popup transparency matches terminal opacity |
| `laststatus` | 3 | One global status bar, not per-window |
| `timeoutlen` | 300 ms | Which-key popup appears after 300 ms |
| `updatetime` | 250 ms | Faster CursorHold for gitsigns and LSP |

Markdown, text, and gitcommit files automatically get `wrap`, `spell`, and `linebreak` via an autocmd. Go files switch to real tabs at 4 spaces.

### LSP Servers

[↑ Neovim](#neovim)

All servers are installed via Nix — not Mason. Configured in `dotfiles/nvim/lua/plugins/lsp.lua` using the Neovim 0.11 `vim.lsp.config` / `vim.lsp.enable` API.

| Language | Server | Notes |
|----------|--------|-------|
| Lua | `lua_ls` | Knows about the `vim` global; configured for LuaJIT |
| Nix | `nil_ls` | Formats with alejandra; auto-archives flake inputs |
| Bash | `bashls` | |
| TypeScript / JavaScript | `ts_ls` | Inlay hints enabled |
| Python | `pyright` | Basic type checking, auto search paths |
| Rust | `rust_analyzer` | Runs clippy on save; inlay hints enabled |
| Markdown | `marksman` | |
| YAML | `yamlls` | Schema store + validation |
| TOML | `taplo` | |
| C / C++ | `clangd` | Background index + missing include suggestions |

All servers share the same `on_attach` (which sets the LSP keymaps) and cmp capabilities. Diagnostics appear inline as virtual text, suppressed while typing in insert mode.

### Matugen Color Sync

[↑ Neovim](#neovim)

When the wallpaper changes, the matugen pipeline regenerates `~/.config/nvim/colors/matugen.lua`. An autocmd in `autocmds.lua` watches this file's modification time on every `FocusGained` and `BufEnter` event, then reloads the colorscheme live — no restart needed and no visible flash.

The colorscheme template lives at `dotfiles/nvim/templates/matugen.lua` — matugen fills in the colors and writes the output.

## 🐚 Zsh

[↑ Back to Contents](#-contents)

**Role:** Default shell for all users on all machines. Migrated from fish.

### Configuration layers

The zsh config is layered across three sources, applied in order:

1. `programs.zsh` in `home.nix` — Home Manager options: autosuggestions, completion, fast-syntax-highlighting plugin
2. `programs.zsh.zsh-abbr.abbreviations` — declarative abbreviations (see [05-maintenance.md](05-maintenance.md) for the full list)
3. `dotfiles/zsh/zshrc` — shared env vars, PATH, fastfetch on open, `nru` function, completion styling; symlinked to `~/.zshrc`

### Integrations

All enabled via `enableZshIntegration = true` in `home.nix`:

| Integration | What it does |
|-------------|-------------|
| starship | Prompt — shows git status, directory, command duration |
| zoxide | Smart `cd` — use `z dirname` to jump to frequently used dirs |
| fzf | Fuzzy history search (`Ctrl+R`) and file search (`Ctrl+T`) |
| direnv | Per-directory environment variables — loads `.envrc` automatically |

### Servers

Servers use `modules/base/server-shell.nix` instead of Home Manager. This module configures zsh with autosuggestions, syntax highlighting, and shell aliases inline. No per-user Home Manager runs on servers.

## 🆘 Troubleshooting

[↑ Back to Contents](#-contents)

### Neovim

| Problem | Fix |
|---------|-----|
| Telescope doesn't find hidden files | `fd` must be installed — it's in `home.packages`. Telescope is configured to use `fd` explicitly |
| Image preview fails (E966 screenpos) | Image rendering is deferred to fix this — if it still fails, restart Neovim |
| LSP not working after nixpkgs update | Check `dotfiles/nvim/lua/plugins/lsp.lua` — the lspconfig 0.11 API uses `vim.lsp.config` / `vim.lsp.enable`; older patterns are deprecated |
| Colorscheme resets after rebuild | `~/.config/nvim/colors/matugen.lua` is written by matugen on the first wallpaper change — trigger a wallpaper cycle to regenerate it |
| Formatter not running on save | Run `:ConformInfo` — confirm the formatter binary is in PATH (all formatters are Nix packages in `home.packages`) |
| which-key popup too slow | `timeoutlen` is 300 ms in `options.lua` — lower it if needed |

### Ghostty / Kitty

| Problem | Fix |
|---------|-----|
| Colors look wrong after wallpaper change | Run `systemctl --user restart wallpaper-color-sync` to force a refresh. Check `~/.local/share/last-matugen-wallpaper` if it keeps reverting |
| Ghostty shaders not loading | Confirm shader files exist in `dotfiles/ghostty/shaders/` and the symlink at `~/.config/ghostty/shaders/` is in place |
